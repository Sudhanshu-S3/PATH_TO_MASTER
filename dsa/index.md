### Data structures and Algorithms

The structure I am following is based on my own understanding, not a fixed
syllabus. Back to [README](../README.md).

#### Sequence

Round 1 covers every topic before Round 2 starts on any of them. Breadth first,
then depth - that way mixed practice can begin early, and each topic gets a
second pass with the others already in place.

Round 1, in this order :

| # | Topic | Roadmap | Questions |
| - | ----- | ------- | ------------ |
| 1 | DP | [dp](dp/entry.md) | [question](dp/practice/index.md) |
| 2 | Graph | [graph](graph/entry.md) | [question](graph/practice/index.md) |
| 3 | Tree | [tree](tree/entry.md) | [question](tree/practice/index.md) |
| 4 | Range Query | [rangeQuery](rangeQuery/entry.md) | [question](rangeQuery/practice/index.md) |
| 5 | String | [string](string/entry.md) |[question](string/practice/index.md) |

Round 2 follows the same order.

Each topic folder holds an `entry.md` with the roadmap and round split, and a
`practice/index.md` where solved problems are logged.

#### NOTE

Splitting a topic into sections is not about memorizing patterns. The point is
to meet each concept from every angle, so the understanding is structured
instead of scattered, and the clarity carries over to problems that do not match
any pattern.
